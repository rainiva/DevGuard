# UI Review Extension

Use this during review of UI-facing work.

## Review Checks

Verify:

1. the new UI follows the product's existing visual system unless a deliberate redesign was requested
2. the hierarchy is clear without unnecessary visual weight
3. major states are present and reachable
4. visible actions are bound to real functionality
5. responsive or desktop-fit behavior was considered
6. text, labels, and empty states are concise and useful
7. target-platform design guidelines were followed where applicable, or the deviation is explicitly justified
8. any platform-guideline deviation does not silently damage accessibility or host consistency

## Findings Focus

Prefer findings around:

- missing states
- broken or fake bindings
- visual drift from the established system
- layout breakage
- inaccessible or confusing interaction
